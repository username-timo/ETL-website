Last updated: May 20, 2026

This file is a simple map of the ETL Next.js project. Use it when you want to know where a feature lives before editing.

## Project Overview

- Client/site: Engineering Trade Links Co. Ltd
- Local folder: `C:\WORK\ETL\ETL_website\etl-nextjs`
- GitHub repo: `https://github.com/username-timo/ETL-website.git`
- Main branch: `main`
- App type: Next.js 15 App Router site deployed to Cloudflare Workers through OpenNext
- Public marketing site: React/Next.js files in `src/app`
- Internal/demo tools: static HTML pages and extracted JS/CSS files in `public`

## Tech Stack

- Next.js 15, React 19, TypeScript
- Tailwind CSS and normal CSS files
- Cloudflare Workers with OpenNext
- Supabase for Auth, Postgres tables, and REST API access
- Brevo for transactional email
- Cloudflare Turnstile for public form spam protection
- Supabase Auth for internal tool login

Note: NextAuth is not the active auth system for the current tool pages. The working internal-tool login is handled by Supabase Auth through `public/shared/etl-auth.js`.

## Do Not Edit These Generated Folders

- `.next/` - generated by Next.js builds
- `.open-next/` - generated by OpenNext Cloudflare builds
- `.wrangler/` - generated by Wrangler/Cloudflare tooling
- `node_modules/` - installed packages

Edit source files in `src/`, `public/`, and root config files instead.

## Auth and Security Notes

- Internal tools use Supabase Auth through `public/shared/etl-auth.js`.
- User roles live in `profiles.role`, mainly `staff` and `management`.
- Public view links do not require login. They rely on unguessable `unique_link` values and Supabase RLS.
- Public request forms use Turnstile before email sending.
- Internal email sends use the logged-in Supabase session token and are rejected by `/api/send-email` if the session is missing or expired.
- The quote reminder endpoint fails closed in production if `CRON_SECRET` is not configured.
- The Supabase anon key is visible in browser JS by design. It must never be treated like a private secret.
- Real secrets such as `BREVO_API_KEY`, `TURNSTILE_SECRET_KEY`, `CRON_SECRET`, and `SUPABASE_SERVICE_KEY` belong in Cloudflare secrets or local `.env`, not public files.
- Before launch, run `supabase-rls-audit.sql` in Supabase and review every anon grant/policy result.
- Public document links should use RPC functions such as `get_public_quotation`, `get_public_lpo`, and `get_public_invoice`, not direct anon table SELECT.
- Public/static pages still use some `innerHTML`; user/database text must be escaped before it is inserted into those HTML strings.

- Quotation generator and invoice pages use the client phone when available.
- Outward LPO uses the supplier/vendor phone when available.
- Inward LPO and public quotation request send to the ETL team number `+256704545163`.
- If no phone number is available, WhatsApp opens the manual contact picker with the message prefilled.
